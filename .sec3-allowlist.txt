# --- UnvalidatedAccount ---
# X-Ray cannot parse Anchor's declarative #[account()] constraint
# macros, so it reports accounts as "unvalidated" even when they are
# fully constrained via PDA seeds+bump, has_one, and explicit
# constraint checks. All accounts in all 14 instruction handlers use
# Anchor constraints for validation. The two UncheckedAccount usages
# (fee_destination in initialize_vault, session_rent_recipient in
# finalize_session) have documented /// CHECK: comments and explicit
# runtime validation. Oracle remaining_accounts are validated in
# oracle.rs by owner program ID and expected feed pubkey.
UnvalidatedAccount
